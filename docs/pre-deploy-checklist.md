# 上云前检查清单

本文档用于本地最终联调完成后、正式部署到 `http://10.10.0.21/` 之前的最后检查。

## 1. 代码与构建

- [ ] 后端编译通过：`python -m compileall backend/app`
- [ ] 前端构建通过：`cd web && npm run build`
- [ ] 本地冒烟通过：`./scripts/smoke-backend.sh`
- [ ] 本地互测清单已至少完整走过一次

## 2. 环境变量

- [ ] 已准备生产环境 `.env`
- [ ] `APP_ENV=production`
- [ ] `AUTH_MODE=token`
- [ ] `AUTH_SECRET` 已替换为随机长字符串
- [ ] `DATABASE_URL` 指向云服务器数据库
- [ ] `CORS_ORIGINS` 包含 `http://10.10.0.21`
- [ ] `UPLOAD_DIR` 指向持久化目录，例如 `/opt/student_service/uploads`
- [ ] `AUTO_CREATE_TABLES=false`
- [ ] `ENABLE_SCHEDULER=true`

说明：

- 邮件网关如未配置，可暂时保持为空，系统仍可通过站内信完成互测主链路。

## 3. 数据与存储

- [ ] 数据库可连接
- [ ] 首次部署已执行：`PYTHONPATH=backend python -m app.seed`
- [ ] 上传目录具备读写权限
- [ ] 已准备至少一个真实模板文件供互测下载
- [ ] 若导入真实学生数据，已确认角色、班级、初始密码列正确

## 4. 本地最终联调结果

建议至少确认以下链路：

- [ ] 无 Token 访问 `/api/session` 返回 `401`
- [ ] 学生账号可正常登录
- [ ] 管理老师账号可正常登录
- [ ] 学生可提交申请
- [ ] 管理老师可审批申请
- [ ] 协同管理者无法越权访问非本范围申请
- [ ] 管理老师可发布通知
- [ ] 学生可查看站内通知与收件箱
- [ ] 学业报告接口返回正常
- [ ] 模板未上传时返回明确错误，而非占位文件
- [ ] 审计导出与申请导出接口可用

## 5. 云服务器部署前置

- [ ] 服务器已安装 Python 3、venv、Nginx、数据库客户端
- [ ] `git` 可正常安装和使用
- [ ] 代码仓库已可拉取到服务器
- [ ] 已规划部署目录，例如 `/opt/student_service/`
- [ ] 已确认 Nginx 可以监听 `80` 端口
- [ ] 已确认 `127.0.0.1:8000` 可供 Nginx 反代

## 6. 部署后首轮验证

部署完成后立即执行：

```bash
curl -s http://127.0.0.1:8000/health
curl -s http://127.0.0.1/health
BASE_URL=http://10.10.0.21 ./scripts/smoke-backend.sh
```

浏览器侧至少确认：

- [ ] 首页可打开
- [ ] 无 Mock 切换入口
- [ ] 学号 + 密码可以登录
- [ ] 学生端可完成一条主链路操作
- [ ] 管理端工作台可打开

## 7. 建议上线顺序

1. 本地完成联调与构建
2. 准备 `.env` 和上传目录
3. 服务器安装依赖并拉代码
4. 初始化数据库与种子数据
5. 启动后端
6. 构建前端
7. 配置 Nginx 反代
8. 运行冒烟脚本与浏览器验证
